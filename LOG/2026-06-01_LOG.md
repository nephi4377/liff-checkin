## 🛠️ 技術實作 (Tech Detail)

### 1. 企業機能分區盤點
*   **Electron Desktop 客戶端**：邊緣端行為視窗標題監測 (15s 採樣)，SQLite 累計工時，Mascot (6種裝束隨機切換) 影像本地快取。
*   **GAS 雲端後端 (backend/)**：
    *   `CheckinSystem`：處理考勤、假勤班表，寫入 Google Sheets，全域快取更新。
    *   `project-console`：日報資料路由，與 Firebase 橋接，利用 LINE 訊息推播。
    *   `core_library`：公用 API（Gemini AI 重點摘要、Dropbox 實體照片物理儲存與預覽連結、LINE Messaging API）。
*   **Web 戰情前端 (CODING/)**：`daily_report.html` 使用前端重度聯集 (Promise.all JOIN) 降低後端運算成本，配合 Waterfall 動態瀑布渲染，對請假者進行 60% 淡化、缺交者紅字警示。
*   **行動端 LIFF**：現場 GPS 打卡，工務照片上傳。
*   **TanxinTools**：BGM v3 備份守護 daemon 巡檢（虛擬保險絲校驗）；本地 Accounting SQLite 記帳 + OCR 發票收據辨識。
*   **投資分析**：Nephi 6 大投資大師評估框架 (巴菲特、查理蒙格、葛拉漢等) 與大師戰情面板、宏觀 (地緣情緒) 4 區間預測。

### 2. 核心營運與生活動線 (Data Flow)
*   **出勤打卡動線**：網頁點擊鎖定按鈕 (防重複寫入) ➔ AppCore IPC ➔ CheckinService ➔ GAS WebApp ➔ 寫入 Sheets ➔ 回傳 JSON ➔ 前端解鎖按鈕並更新今日出勤。
*   **施工日報上傳動線**：工務手機 LIFF ➔ project-console ➔ Dropbox API 物理儲存並生成連結 ➔ Gemini API AI 重修摘要 ➔ 寫入 Sheets ➔ LINE 推播。
*   **戰情室呈現動線**：`daily_report.html` ➔ Promise.all 拉取考勤、日報 ➔ 瀏覽器本地對齊 Join ➔ 未交者/請假者 (淡化60%) 邏輯標示 ➔ Waterfall 引擎分組顯示。
*   **備份巡檢動線**：BGM Daemon (run_daemon.bat) ➔ 每15分鐘執行 ➔ 遠端比對、檢查 `fuse` 保險絲 ➔ 正常則安靜記錄；異常寫入 alerts.json，觸發 charts 生成告警網頁。

### 3. 三大安全施工規範彙整
*   **防併發鎖定**：UI 點擊按鈕後必須 Disabled 直至 API 回傳。
*   **程序 relaunch**：小助手熱更新補丁後必須使用 `app.relaunch()` 重啟，避免 Electron 在軟重載下因 IPC 二次註冊崩潰（Attempted to register a second handler）。
*   **標準部署流程**：強制執行 `backend/upload.bat` 自動完成 robocopy 本地備份 ➔ git push ➔ 觸發 GitHub Actions 自動 clasp 部署。嚴禁人工手動 `clasp push`。
