## 各問題根因（對照程式）

### 1. 中和碁電整筆失敗

- 標已匯款會先「寫收支」再改狀態。
- 寫收支時，分攤列若有**負數**，舊驗證當成「金額不合法」直接失敗。
- 失敗就**不會**標已匯款、也**不會**生 `vpay_` → 與回報一致。
- 案號 999、缺 vpay_ 是結果不是主因；**負數分攤**才是擋點。

### 2. 行政 LINE 推兩次

可能疊加：

1. 廠商綁定表有 UID，主檔又留舊 `line_user_id` → 舊程式**兩邊都推**。
2. 同一收件人對應**多筆**廠商匯款 → 一筆一則，看起來像連推兩則。

### 3. 官 LINE 全沒出

推播依綁定註記選「會計／官方」token。常見：

1. Script Property **未設** `OFFICIAL_LINE_CHANNEL_ACCESS_TOKEN`（或舊名 HUB token）→ 官方通道直接略過。
2. 綁定註記沒被辨成 official → 誤用會計 token 推官方 UID → API 失敗。
3. 廠商根本只綁會計通道 → 官 LINE 本來就不會有訊息。

### 4. 五筆都沒進收支明細？

- 程式順序：**寫收支成功 → 才標已匯款並寫 `vpay_`**。
- 四筆已有 `vpay_`／已匯款 → **理論上收支列應存在**（在**收支明細試算表**當月分頁，不是會計主檔請款表）。
- 若真的搜不到：可能看錯表／分頁，或當下寫入異常；修正後「有 ingest_id 但無列」會自動補寫。

### 5. 毛利有沒有計入？

- 毛利在「寫收支成功之後」同一條後處理寫入（依分攤案號）。
- **沒寫進收支＝通常也沒寫毛利**；有收支列才可能有毛利列。
- 與收支**未完全脫鉤**，但是兩個試算表；查毛利要開案件毛利分頁搜同一 `vpay_`。

### 6. 毛利表欄位左移／缺案號（2026-07-20 追加）

**現象**：`明細_X00` 表頭是「案號｜日期｜收支｜…」，今日 `vpay_`／`vpfee_` 列卻從「日期」寫起，整排左移一格。

**根因（不是多路徑各寫一套）**：

| 路徑 | 函式 | 是否缺案號（修前） |
|------|------|-------------------|
| 標已匯款／LINE 記帳 → 寫收支後 | `LedgerPostIngest` → `appendMarginLineFromIngest_` → `appendOneMarginLineFromIngest_` | ✅ 缺（今日壞列） |
| 手動加列 | `handleMarginAddLine_` | ✅ 缺 |
| 歷史匯入／backfill | `queueMarginBackfillLine_` 等 | ✅ 缺 |
| 系統基本費用／工資 | `upsertSystemMarginLine_` | ✅ 缺 |
| 客戶收款入毛利 | `cfWriteReceiptIncomeToMargin_` | ✅ 缺 |

以上**全部共用** `MARGIN_LINE_HEADERS_`（舊為 8 欄、無案號）。表／SPEC（2026-07-18）已改「首欄＝案號＋`明細_X00`」，**程式常數與寫入列從未對齊**——不是「修了一條忘了另一條」，而是**規格／實表改了、寫入端整包沒跟上**。較早正確列多半是併表／補案號時寫進的；今日批次是遷移後首批大量 `appendRow`，才明顯露出左移。

**為何像「修過又復發」**：7/18 規格與試算表已換成有案號的格式，看起來像修好了；雲端跑的仍是舊 8 欄寫入 → 新列一寫就左移。

## 7. 毛利只寫一列／項目沒對上／匯費掛案（2026-07-20 第三輪）

對照今日批次截圖與分攤列（僑美 4 案、周龍泉 2 案、明豪多案等）：

### 現象 vs 應有

| 現象 | 應有 | 根因 |
|------|------|------|
| 多案分攤毛利只見**一列**（如僑美只見 729／13020、周龍泉只見一筆 12000） | 依分攤**一案一列**（收支仍 1 筆 `vpay_`） | 7/18 併入 `明細_X00` 後，多案落在**同一分頁**；即時寫入仍用「同一 ingest_id＝已寫過」去重 → 第二筆起被略過。歷史回填路徑早已用 `ingest_id:案號`，即時路徑沒跟上 |
| 項目名稱不像分攤品名 | 各列＝`廠商／該分攤品名` | 只留下第一筆時，其他案的品名根本沒寫入；再加上左移後「項目名稱」欄錯位，看起來更對不上 |
| 匯費有的有、多案常沒有 | 每筆匯款另寫 `vpfee_` $30，毛利掛**金額最大**那案 | 匯費案號用了「773 等4案」這種彙總標籤 → 毛利拒絕寫入。單案（如協進 777）才能寫進。舊「扣匯費」規則本就取最大金額案，寫匯費列時遺失 |

與「缺案號左移」**同一遷移脈絡、不同機制**：左移＝少寫案號欄；只寫一列＝同分頁 ingest 去重過嚴。兩者疊加造成今日批次毛利很難用。

## 建議使用者怎麼驗（部署後）

1. 待匯款頁對**測試筆**或中和碁電（部署後）再「標記已匯款」。
2. 收支試算表當月分頁（民國年月）搜 `vpay_` → 廠商全額 1 列；搜 `vpfee_` → 匯費 $30。
3. 案件毛利 `明細_X00`：同一 `vpay_` 應對分攤有**多列**（各案號＋品名＋金額）；`vpfee_` 應在**金額最大**那案；**A 欄＝案號、B＝日期**。
4. LINE：同一窗口多筆只收**一則**合併通知；官方綁定者應收到官方帳號推播（token 正確時）。

## 今日補帳手動步驟（部署前也可先查）

1. **先查**：收支試算表 7 月分頁搜四筆已匯款廠商名或備註裡的 `vpay_…`。
   - **找得到** → 不用補收支；再查毛利即可。
   - **找不到** → 記下請款 `payment_request_id`／`vpay_`，部署本修後對該筆再按一次標已匯款（會走補寫）；或請資訊手動補列。
2. **中和碁電**（仍未匯款）：部署後直接再標一次即可（負數分攤已放行）。部署前若急：先拿掉或改寫該筆負數分攤再標（不建議長期）。
3. **補推 LINE**：部署後若只需通知、狀態已正確——可暫時不要重標；改由人工私訊，或等下次批次。若重標，已匯款筆會被略過，**不會**再推。需補推時請資訊用既有推播／手動 Push。
4. **官 LINE**：先確認 Property token，再確認綁定通道；否則程式修好也推不出去。

### 今日毛利左移列怎麼修（試算表手動）

1. 開**案件毛利**試算表，相關 `明細_X00` 分頁。
2. 搜尋今日 ingest：`vpay_`、`vpfee_`（或約第 415 列起、來源=`自動`、備註=`vendor_payment`／`vendor_payment_fee`）。
3. 辨識錯列：A 欄是**日期**（如 `2026-07-20`），不是案號。
4. 對這些列：在 A 欄**插入空白儲存格並右移**，再於 A 欄填入正確**案號**（可對請款分攤或項目名推斷）。
5. 修完後 A＝案號、B＝日期、…、H＝ingest_id、I＝備註。
6. **勿**在未修左移前列時重跑寫入（舊程式用 ingest 對不到錯欄，可能再長一筆正確列變成重複）。
