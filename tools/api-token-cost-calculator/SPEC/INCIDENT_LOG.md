# API Token Cost Calculator — 失敗與復原事件記錄 (Incident Logs)

本文件用以記錄開發 `api-token-cost-calculator` 過程中的重大失敗、功能損毀、或 Bug 修復事件，以確保後續協作不再犯相同錯誤。

## 2026-05-31 00:10 | index.html 歷史還原導致載入日誌 UI 元素丟失及 JS 型態比對錯誤

### 1. 事件經過
*   **背景**：先前已實作的「輸入案號自動載入施工日誌與照片預覽」功能，總監反應失效、按鈕消失。
*   **起因**：
    1. **UI 元素丟失**：先前可能因 `git restore` 或程式碼分支還原，導致 `index.html` 內的 `<button id="btn-load-backend-logs">` 及 `<div id="backend-logs-group">` 容器被意外清除。
    2. **JavaScript 型態不合**：下拉選單變更時傳入 `applyBackendLog(this.value)` 的引數是字串（如 `"0"`、`"1"`），這在某些 JS 執行環境中會導致 `backendLogsList[index]` 取出 `undefined` 而使日誌載入崩潰。
*   **結果**：總監無法在測試工具網頁上點擊「載入日誌」，亦無法看見已發布日誌的下拉選單。

### 2. 復原與修復行動
*   **解決方案**：
    1. **補回 HTML**：將「載入日誌」按鈕與「施工日誌選擇」下拉選單重新加入 `index.html` 的 `live-project-id` 輸入區。
    2. **JS 防禦性轉型**：在 `applyBackendLog(index)` 頂部加入 `parseInt(index, 10)`，防範 string 型態所致的快取陣列對照失效。
*   **修復結果**：成功。日誌與照片載入功能完好恢復。

### 3. 經驗教訓與防範雷區
1.  **合併/還原操作後全面檢查 UI 完整性**：在進行任何還原或版本修復後，必須全面比對 HTML 控制元件是否依然健全。
2.  **型態防禦性設計**：凡是由 HTML Element (如 `select.value`、`input.value`) 傳入 JavaScript 做為陣列 index 使用時，務必在接收端先強制進行數字轉型 (`parseInt`) 並作 `isNaN` 檢查，杜絕隱性 runtime 報錯。

## 2026-05-31 00:20 | Google Drive 縮圖限制 CORS 代理下載失敗之全自動繞過與離線壓縮

### 1. 事件經過
*   **背景**：總監已將 Google Drive 資料夾設定為「知道連結的任何人皆可檢視」，但系統依然提示 `⚠️ 限制 (CORS)`，並要求手動下載拖拉。
*   **起因**：Google 的 `thumbnail` 縮圖 API 端點有較嚴格的安全憑證機制，即使檔案是公開的，第三方代理伺服器在沒有瀏覽器登入 Cookie 時也會被要求登入（回傳 403 / 520）。
*   **結果**：即使檔案是公開的，背景代理伺服器下載縮圖依然失敗，無法實現 100% 全自動轉 Base64 送給 AI。

### 2. 復原與修復行動
*   **解決方案**：
    1. **引入公開下載 API (`uc?id=...`)**：在 `downloadImageAsBase64WithFallback` 中加入雙重網址輪詢。當 `thumbnail` 下載失敗時，改下載公開的檔案原始檔（如果是公開檔案，代理伺服器就 100% 能下載）。
    2. **離線 Canvas 壓縮**：下載圖片的 Blob 後，在前端建立一個離線的 `<canvas>`，將其等比例壓縮至最大寬度 `800px`（品質 `0.85`）。此時因為 Blob 已經在同源環境下，**完全不會觸發瀏覽器的安全畫布污染限制**，且依然維持了低 Token 的消耗。
*   **修復結果**：成功。只要檔案為公開檢視，全部照片在背景即可 100% 全自動轉換為 Base64，不需任何手動拖曳動作。

### 3. 經驗教訓與防範雷區
1.  **區分 HTML 載入與 JS Fetch 請求**：`<img>` 標籤自帶 Cookie 容易成功，但 JS 代理下載在無 Cookie 下容易受限。設計時應兩者兼顧。
2.  **前端本地 Canvas 壓縮繞過 CORS**：如果跨來源圖片不允許 Canvas toDataURL，可以先利用代理伺服器將圖片以二進位（Blob）下載回本機，再畫上 Canvas。這時圖片已無跨域問題，能 100% 成功進行尺寸調整與轉碼。
