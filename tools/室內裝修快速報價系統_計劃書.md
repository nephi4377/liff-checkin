# 室內裝修快速報價系統 — 產品計劃書

> **版本**：2026-04-26 v0.5（主線：方案「菜單」逐項數量規則，見 §4A、SPEC §1.4）  
> **對應技術規格**：`CODING/SPEC/14_INTERIOR_RENOVATION_QUOTATION_SYSTEM_SPEC.md`（**§4 必要參數與組態界定**、§5 FR、§6 資料模型）  
> **參考**：市面「現場選工項、一鍵匯出」類工具之常見賣點；下列「三步驟」對照來自同類產品行銷／教學型折頁之**公開流程描述**（僅萃取功能與 UX 結構，不涉及商標或原始素材）。

---

## 1. 為什麼要做

室內設計／裝修業在**現場丈量或初談**時，需要快速產出「可給客戶看、可給內部對價」的估價範圍。  
現有 CODING 生態裡，**LayoutPlanner**（見 `SPEC/04_INTERIOR_DESIGN_TOOL_SPEC.md`）擅長「圖上放物件、依幾何自動算價」；本計劃補上另一條路：**不畫圖、以空間＋工項清單為主**的快速報價，兩者可並存或日後串接。

---

## 2. 誰用、在什麼時機用

| 角色 | 典型情境 |
|------|----------|
| 設計師／業務 | 客戶現場或視訊，15–30 分鐘內要「區間報價」或「項目式明細」 |
| 公司內部 | 統一工項與單價版本，減少每人各用各表 |
| （選配）工地／專案 | 與「工程進程、報告」同一品牌下擴充時再接 |

首版以**單一公司內部＋平板／手機網頁**為主，不強求第一天就做到完整客戶端入口。  
介面節奏建議與下節「三步驟」一致，降低業務學習成本（與市面已教育過使用者的流程對齊）。

---

## 3. 參考市售流程對照：三步驟、報價超輕鬆

以下為**同類產品常見主流程**（濃縮自公開文宣），作為我們規格與畫面的對照標竿；實作優先級仍以 **§7 MVP** 為準。

| 步驟 | 使用者在做什麼 | 介面與能力（參考項） |
|------|----------------|----------------------|
| **一** | 設定有哪些空間、各多大 | **網格式挑空間**（客廳、主臥、廚房等圖卡＋加入）；輸入**坪數**或**長寬等尺寸**；**坪與㎡等換算**由系統協助，減少手算 |
| **二** | 每個空間要裝哪些項目 | **依空間切換**看工項；大類（拆除、天花、地坪、櫃體等）再進**細項卡片**；支援**加購／特殊工法**與文字細項；單價可依**空間／區域**帶入不同價本（若公司政策需要） |
| **三** | 確認總價、調整、給客戶 | **總覽表**：各行名稱、單價、數量、小計；另塊處理**設計費**、**工程服務費**（常見為依小計或工程款之**百分比**，可改）；**稅額**（例如固定稅率％）自動算；尾段可**自由增刪行**再定案；**存檔**＋**一鍵導出**（PDF／表格式） |

**我們可刻意對齊的差異化句**：靈活空間配置、工項齊全、空間區域計價、尾段自由增減、設計費與工程費公式化＋稅額一併呈現。

## 4. 產品要長什麼樣（功能清單，與 §3 呼應）

### 4A. 主線（客戶先認「單位價」— 參考報價）

1. **房型與總坪**：選模板（如三房二廳）、輸入**總坪數（不含公設）**。  
2. **系統推估**：依管理員設定分配陽台、浴廁、客廳+廚房、各臥**坪數**（客廚占比可設區間並帶**隨機性**）；臥室間依主臥／次卧倍率；再以**長方形比例**（如 4:6 或 6:4）反推各區**概略長寬**，供櫃體尺寸粗估。  
3. **常見項目 + 方案菜單**：勾選天花板、冷氣包管、窗簾盒、衣櫃、鞋櫃、收納櫃、**餐櫃**、**電器櫃** 等；客戶選**其中一套方案**（可擴充多套）。**每一品項數量**由該方案之規則決定（例：主臥衣櫃捉**短邊台尺**× 精簡 60%、收納滿短邊；鞋櫃精簡 3 尺／收納 4 尺；天花板坪數＝客廚+臥室合計**向上取整**；冷氣包管預設**客廚短邊尺**）— **不是**整單再乘同一個百分比。  
4. **免責與目標**：輸出明確標示「僅供參考」；商業目標是客戶先理解並認同公司**單位價格**級距，再進設計細談（需求決定總價）。

**管理員**：獨立設定頁維護坪數推估參數、各品項**單價**，以及**方案菜單 JSON**（可匯入匯出、做多套方案）；試作見 `tools/QuickRenovationQuote_v2.html`。

### 4B. 支線（清單式價本勾選 — 與 §3 三步驟對照）

1. **步驟一 — 空間與面積**：案名、空間清單、坪數或尺寸、單位換算、備註。  
2. **步驟二 — 工項**：依空間勾選、分類／子分類、數量與計價規則、加購與特殊工法（文字或價本列）。  
3. **步驟三 — 總覽與費用**：明細小計、設計費、工程服務費、稅、總計；可於本頁增刪調整後再定稿。  
4. **匯出與留存**：存檔、一鍵導出（PDF／試算表類）；公司抬頭、日期、版本。  
5. **歷史與管理**：列表、複製、草稿／已定稿（或版本號）。  
6. **（第二階段）** 共享、工程進程、統計報表 — 與「專案主控／施工回報」整合時另開介面與 SPEC 子節。

## 5. 必要參數（須先拍板；逐項定義在 SPEC）

下列**類別**在寫程式前就要由產品／會計界定預設值與可否覆寫；**表格、型別、預設、合法範圍**一律以 **`SPEC/14_INTERIOR_RENOVATION_QUOTATION_SYSTEM_SPEC.md` §4** 為準，本計劃書不重複維護。

| 類別 | 要決定什麼（白話） |
|------|---------------------|
| **面積／單位** | 1 坪換多少㎡、小數幾位怎麼捨入、畫面要不要同時顯示坪與㎡；若用長×寬換算，單位與「手改坪數誰優先」 |
| **金額捨入** | 每一行小計、小計加總、設計費／工程費／稅，是否分別捨入、捨到哪一位（通常整數元） |
| **總價堆疊** | 設計費與工程服務費用「固定」還是「依小計％」、％的基礎是誰；稅要不要開、稅基是否含兩種費用；自訂行／加購／特殊工法算不算進「工程小計」 |
| **上限與折扣** | 一張報價最多幾空間、幾行；行折扣用％或金額擇一；定稿要不要鎖價本版本 |
| **匯出** | 預設匯出 PDF 還是表格式、檔名規則、抬頭資料從哪來 |

**流程**：凍結 **§4** → 欄位寫入 **`PROJECT_DATA_DICTIONARY.md`** → 再開 API／畫面實作。

## 7. MVP 範圍（建議第 1 個可上線版本）

**一定要**

- 新增／編輯一筆報價：多空間、每空間坪數（或尺寸換算為面積）、備註。  
- **三步驟導覽**（可簡單步驟條＋下一步）：空間 → 工項 → 總覽。  
- 工項庫：分類、子分類、名稱、單位、單價、計價方式（固定式／每坪／每米等至少兩種）。  
- 依空間勾選工項、改數量、**行小計**與**工程明細小計**即時更新。  
- **總覽頁**：設計費、工程服務費（至少支援「依某基礎金額之百分比」或固定金額擇一）、**稅率與稅額**、報價總額（計算順序與參數見 `14_` SPEC **§4、§5**）。  
- **尾段可增刪明細行**（含自訂一行：名稱＋金額或數量×單價）。  
- **存檔**＋匯出至少一種：**PDF** 或 **可下載的表格式檔**（二選一先做好再擴充）。  
- 報價列表＋開啟編輯＋複製。

**有則加分、可列 P1（原型後再決）**

- 空間以**圖卡網格**挑選（視覺接近參考折頁）。  
- **加購項／特殊工法**獨立價本區或標籤。  
- 同一工項依**空間類型**帶不同預設單價（區域計價）。

**先不做（列入 Phase 2）**

- 與圖面雙向連動、自動從平面圖算坪數。  
- 客戶線上簽名、線上付訂金。  
- 完整權限 RBAC、多公司租戶。

## 8. 階段規劃（粗估）

| 階段 | 目標 | 產出 |
|------|------|------|
| P0 企劃 | 對齊名詞、流程、MVP | 本計劃書 + `14_` SPEC |

工時受「工項庫誰維護、匯出範本設計、是否上 LIFF」影響大，不在本檔寫死人天；排程以 P1→P2 垂直切片為原則。

## 9. 風險與假設

- **工項與單價**若無負責人維護，系統再快也會失去信任；需指定「價本」更新節奏。  
- **法律與合約**：本工具產出為「估價參考」或「報價單」需由公司自行定義文案免責。  
- **離線現場**：若工地無網路，需評估離線草稿（複雜度高，建議 MVP 先假設 4G 可用）。
