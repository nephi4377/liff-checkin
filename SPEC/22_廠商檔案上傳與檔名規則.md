# 廠商檔案上傳與檔名規則

> 會計模組：行政以上（≥2）上傳廠商報價單、合約等到 Google Drive，並寫入試算表索引。  
> 後端 API 細節見 `backend/accounting-gas/SPEC/VENDOR_DOCUMENT_SPEC.md`。

---

## 1. 功能與入口

| 項目 | 說明 |
|------|------|
| 頁面 | `modules/accounting/payment_request.html`（預設「存檔歸檔」模式；`vendor_docs.html` 轉址至此） |
| 選單 | 會計系統 → 日常款項 → **廠商存檔或待付款** |
| HUB 深連結 | `#/accounting/vendor-docs`（存檔模式）或 `#/accounting/payment-request` |
| 模式 | 頂部切換：**存檔歸檔**（預設）／**待付款請款**；OCR 可建議切換，不自動跳轉 |
| 檔案類型 | 兩模式皆支援圖片、PDF、Word、Excel 等（OCR 僅圖片／PDF）；存檔最多 30 檔、請款最多 20 檔 |
| 權限 | ≥2（行政）；不可改廠商主檔 |
| API | `vendor_doc_submit`、`vendor_doc_list` |

## 2. 檔案名稱設定規則

上傳後的名字由**後端自動產生**，使用者無法自訂 Drive 檔名；表單「備註」只寫試算表，不會出現在檔名裡。

### 2.1 存到哪個資料夾？

**現行（2026-07-16 起，已啟用舊樹路徑）**

```
~裝潢建材廠商 / {工種} / {廠商名稱} / {檔案}
```

- **工種** = 廠商主檔 `trade_category`（上傳不另選）。
- **廠商名稱** = 廠商主檔 `name`（與廠商名冊、待付款流程相同）。
- 若廠商已有 `drive_folder_id`，直接寫入該夾；否則依工種／廠商名在舊樹下自動建立。
- 檔案放在該廠商資料夾的**根目錄**（不另開 `報價單/` 子資料夾）。

**備註**：未啟用舊樹旗標時，後備路徑仍為 `會計廠商檔案／{廠商名稱}`（遷移前舊制）。細節見 `backend/accounting-gas/SPEC/VENDOR_DOCUMENT_SPEC.md`、`ACCOUNTING_MASTER_DATA_SPEC.md` §3.3。

### 2.2 Drive 上的檔案怎麼命名？

**公式**

```
{日期時間}_{檔案類型}[_{案號}]_{序號}_{原始檔名}
```

| 片段 | 說明 | 例子 |
|------|------|------|
| 日期時間 | 上傳當下，台北時間 | `20260716_143052` |
| 檔案類型 | 表單必選：報價單／合約／型錄／規格書／其他 | `報價單` |
| 案號 | 表單有填才加 | `752` |
| 序號 | 同一次傳多個檔：01、02、03… | `01` |
| 原始檔名 | 你選的檔名（會去掉不合法字元） | `廚櫃報價_v2.pdf` |

**實際例子**

| 你做的事 | 存進 Drive 的檔名 |
|----------|------------------|
| 上傳 `廚櫃報價_v2.pdf`，類型「報價單」，案號 752 | `20260716_143052_報價單_752_01_廚櫃報價_v2.pdf` |
| 同一次再傳一個 Excel | `20260716_143052_報價單_752_02_明細.xlsx` |
| 上傳合約、沒填案號 | `20260716_150000_合約_01_合作合約.pdf` |
| 手機現場拍照 | `20260716_143052_報價單_01_photo.jpg` |

**其他規則**

- 原始檔名最長 60 字；`image.jpg` 這類預設名會改成 `photo.jpg`。
- 若資料夾裡已有同名檔，會自動加 `_2`、`_3` 避免覆蓋。
- 禁止 `.exe`、`.bat` 等可執行檔；單檔最大 10 MB。
- **訂編**、**備註**不會出現在 Drive 檔名，只記在試算表方便查詢。

### 2.3 試算表記什麼？

分頁：**廠商檔案索引**（entity：`vendor_document`）

| 欄位 | 說明 |
|------|------|
| `filename` | Drive 上的**完整檔名**（§2.2 產生的那個） |
| `doc_type` | 檔案類型 |
| `project_no` / `order_no` | 案號、訂編 |
| `title` | 備註 |
| `drive_url` | 點開檔案的連結 |
| `uploaded_at` / `uploaded_by` | 何時、誰傳的 |

## 3. 表單欄位

| 欄位 | 必填 | 說明 |
|------|------|------|
| 廠商 | 是 | 從名冊快選 |
| 檔案類型 | 是 | 報價單／合約／型錄／規格書／其他 |
| 案號 | 否 | 有填會進 Drive 檔名 |
| 訂編 | 否 | 只進試算表 |
| 備註 | 否 | 只進試算表 |
| 檔案 | 是 | 多檔、拖放；手機可拍照、相簿或選檔案；最多 30 檔 |

### 2.4 選用 AI 辨識（非每次上傳）

1. 僅**圖片／PDF** 可勾「辨識」
2. 按 **「辨識勾選的檔案」** → `vendor_doc_ocr_analyze`（不建單、不上傳）
3. 可 **「套用至表單」** 帶入廠商、案號、類型、備註
4. 按 **上傳** 時，只有曾辨識的檔案會寫 `ocr_json` 至試算表

### 2.5 財務刪除檔案（≥4）

- 「最近上傳」列表每筆有 **刪除檔案** → `vendor_doc_deactivate`
- Drive 檔案移入 **Google 垃圾桶**（可還原）；試算表 `is_active=否`、列表不顯示

## 4. 程式對照

| 白話 | 程式對照 |
|------|----------|
| 產生 Drive 檔名 | `buildVendorDocumentDriveFilename_`（`VendorDocumentModule.js`） |
| 上傳並寫索引 | `handleVendorDocSubmit_` → `saveVendorDocumentFiles_` |
| 查最近上傳 | `handleVendorDocList_` |
| 前端送出 | `AccountingApi.vendorDocSubmit` |
| 檔案類型選項 | `AccountingMasterData.vendor_doc_types` |

---

## 5. 驗收

1. 行政帳號開頁面 → 選廠商 → 傳 PDF + 拍照 → 成功。
2. 開啟該廠商 Drive 資料夾，檔名符合 §2.2。
3. 試算表「廠商檔案索引」有對應列，`filename` 與 Drive 一致。
4. 頁面「最近上傳」可依案號、類型篩到剛傳的檔。
