# 添心系統擴充與 AI 導入開發備忘錄 (修訂版)

本備忘錄記錄了添心系統後續擴充與 AI 導入的七大核心功能規劃，作為後續系統設計與開發之依據。

---

## 1. 施工照空間視覺定位系統
* **核心邏輯**： 專案開工前，先上傳「施工前現場照片」與「平面配置圖」，並由系統或 AI 擷取各空間特徵值（例如：樑柱位置、窗戶開口、特殊管線配置等）。
* **應用場景**： 後續施工期間，師傅上傳任何現場照片，AI 能自動比對特徵值，判定這張照片是屬於「主臥」、「客廳」還是「玄關」，解決照片堆疊造成的管理混亂。
* **2026-06-24 再確認**：開案階段建立「空間指紋」→ 施工期每張照帶信心度比對；兩間長很像的房要人工確認或加參考照；施工後外觀變了仍可比對結構錨點（窗、樑、管線走向）。
* **開發規格**：見 [`20_施工照空間視覺定位規格書.md`](20_施工照空間視覺定位規格書.md)（v0.1）

## 2. 專案工作區（Project Workspace）介面重構
* **現狀痛點**： 目前檔案與回報僅依「上傳時間」序列頂排序，專案一多、時間一長，閱讀與檢索極為不易。
* **改進目標**： 重新設計呈現方式，提升資訊可讀性（如：改為空間維度、工種維度、或看板式呈現），打破單一時間線的限制。

## 3. AI 視覺化施工進度分析與排程推薦
* **核心邏輯**： AI 不只讀文字，更直接分析施工照片（例如：辨識出輕隔間封板已完成、系統櫃板材已進場）。
* **應用場景**： AI 自動判定該專案當前的實際進程，並主動向設計總監回報，同時根據現況推薦或微調下一步的工程排程。

## 4. 快速報價系統完善化
* **改進目標**： 深化現有的快速報價模組，使其能更精準、快速地根據平面配置與需求生成初步預算書，縮短與客戶前期溝通的週期。

## 5. 官方 LINE 回覆補助擴充程式
* **核心邏輯**： 開發一款適用於 LINE 官方帳號網頁版的瀏覽器擴充功能（Extension）。
* **應用場景**： 當使用者（客服或助理）在將訊息發出時，先透過 AI 檢查語氣與專業度是否合適，並能針對客戶的提問即時提出優化的回覆方案。

## 6. API Token 成本計算與測試網頁
* **核心邏輯**： 配合專案系統導入 AI 的計畫，獨立設計一個網頁，透過 API 的方式去測試每次調用的 Token 花費。
* **應用場景**： 精準計算並記錄每一次 Prompt 的 Input/Output Token 數與美金/台幣花費，作為未來將 AI 完善導入「會計系統」與專案成本核銷 the data foundation.
* **避雷與大規模圖片上傳優化方案 (Files API)**：
  為避免多張高畫質施工照轉 Base64 導致 HTTP 請求體積過大（觸發 20MB 請求大小上限而失敗），在大規模上傳或超大高畫質圖檔情境下，系統必須棄用 `inlineData`，改採兩階段架構：
  1. 後端先調用 **Gemini Files API** (`https://generativelanguage.googleapis.com/upload/v1beta/files`) 將照片或圖面 PDF 上傳至 Google 檔案暫存空間。
  2. 取得回傳的 `fileUri` 後，於 `generateContent` 請求中以 `fileData` 的形式引用該 URI，以確保主請求 Payload 極小化且維持傳送高可靠度。

## 7. 私有化/本地自定 LLM 安裝與整合項目
* **核心邏輯**： 於公司內部伺服器或專屬主機上，自建與部署開源大語言模型（如 DeepSeek、Llama 3 等），並對接 API 提供給添心系統內部使用。
* **應用場景**： 徹底解決裝修合約、業主個資、報價單敏感數據上傳至公有雲的安全隱憂。並在大量進行工法稽核與日誌自動生成時，達成零 API Token 成本的長期運行優勢。
* **實現方案 (技術選型與步驟)**：
  1. **推理引擎選型**：採用開源且輕量化的 **Ollama** 作為本地 LLM 推理引擎，簡化部署與維護工作。
  2. **模型規格配置**：
     * *輕量與高性價比*：選用 `DeepSeek-R1-Distill-Llama-8B` 或 `Llama-3.1-8B-Instruct`。
     * *高級工程推理與合約稽核*：於專用工作站運行 `DeepSeek-R1-14B` 或 `32B` 以提升合約法規與工法檢查之準確度。
  3. **系統對接與介面**：利用 Ollama 內建相容於 OpenAI API 規範的伺服器端點 (`http://[本地伺服器IP]:11434/v1/chat/completions`)，後端不需修改大量代碼，僅需更換 Base URL 與 API Key 即可無縫切換公有雲與私有雲。
  4. **硬體部署建議**：於公司辦公室配置一台搭載專用顯示卡（如 RTX 4060 Ti 16G 或 Mac Studio 高記憶體版）之主機作為內部 AI 推理中樞，不需租用昂貴的雲端 GPU 實例。
