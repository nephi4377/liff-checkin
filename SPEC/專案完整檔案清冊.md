## 9. CheckinSystem 打卡系統後端

**專案位置**：`backend\CheckinSystem`（Google Apps Script）

| 檔名 | 用途 |
|------|------|
| **WebApp.js** | GET/POST 入口、路由 |
| **CheckinLogic.js** | 打卡、假勤核心 |
| **EmployeeLogic.js** | 員工與權限 |
| **ScheduledTasks.js** | 排程、報表、封鎖/清理等 |

> 前後與專案主控（`project-console`）**不同子專案**；專案／回報**前端**在 `modules/projects`，**後端**在 `project-console`（GAS）— 行為以對應 SPEC 與線上腳本為準。

**依賴關係（精簡）**：

```
外部核心函式庫
    ↓
CheckinSystem
    WebApp.js → CheckinLogic.js / EmployeeLogic.js / ScheduledTasks.js / SiteLogic.js …
```

**部分 GET/POST 關鍵字（節錄，表名略）**：

| 行為 | 方法 | 用途（摘要） |
|------|------|----------------|
| checkin | GET | 打卡、GPS/範圍驗證 |
| attendance_api 等 | GET | 出勤查詢、Hub 資料、班表、報表 |
| process_site_form 等 | POST | 案場/排班/員工維護 |

（完整 API 以 `WebApp.js` 與場內實作為準。）

## 10. 使用與維護要點

1. **新增或改變** iframe 模組路由：同步維護 **`spa/app.js`** 之 `routes` 與 **本檔**（含 **§5.4** 與導讀敘述）。  
2. **變更模組行為/欄位**：在對應 `SPEC/*_SPEC.md` 與本檔的「**檔名／路由**」一節**擇一或併**更新，避免兩邊敘事打架。  
3. **大檔/封存**勿誤佈到 GAS：以 `.claspignore` 與專案文件為準。

---

> **重要**  
> 專案目錄、路由、與實體路徑若與**本檔**衝突，**以倉庫內實體與 `spa/app.js` 為准**，並**回寫**本清冊或對應 SPEC。  
> 不複製雲端金鑰、LIFF ID 等到本清冊；**機密**留在各模組的設定區（如 `reportV3.html` 的 `__REPORT_V3_CONFIG__`）。
