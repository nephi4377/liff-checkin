# 專有名詞白話對照（給對話與文件用）

> **用途**：助理或文件必須用到技術詞時，優先用這裡的說法；表內沒有的詞，**第一次出現請在括號內加一句白話**，並可補一行到本表。

| 專有名詞 | 白話意思（可照抄給使用者看） |
|----------|------------------------------|
| **API** | 程式跟遠端服務「要資料或下指令」的約定接口（像點餐單上的編號）。 |
| **CDN** | 放在網路上的現成套件倉庫，開網頁時會去下載圖示、樣式等（需連網）。 |
| **Commit** | 把這一版變更存進版本紀錄，像存檔＋寫一句備註。 |
| **PR（Pull Request）** | 請別人審核「要不要合併這一版程式」的申請單。 |
| **SPEC／規格** | 事先寫好的「應該長怎樣、怎麼用」說明書。 |
| **Repository／Repo** | 整包專案程式與紀錄放在一起的資料夾（通常含 Git）。 |
| **瀏覽器** | Chrome、Edge、Safari 等用來開網頁的程式。 |
| **iframe** | 網頁裡嵌的另一個小網頁畫面（像子視窗）。 |
| **路由（Route）** | 網址 `#/xxx` 對應要開哪一頁的對照表。 |
| **OpenCV** | 做影像辨識、校正等運算的程式庫（常需連網下載）。 |
| **透視校正** | 把斜著拍的矩形（例如圖紙）拉成「正正的長方形」的運算。 |
| **Bounding box／外接框** | 剛好包住選取範圍的「不旋轉的長方形邊界」。 |
| **案場** | 這一個工地／這一戶裝修專案的現場與相關資料（地址、進場方式等）。 |
| **串流留言／留言串** | 針對同一則進度或同一則留言，可以一層層回下去，像社群軟體的討論串。 |
| **客戶進度頁** | 給屋主看的專用畫面，通常只能看自己家、且多半是已公開的進度。 |
| **語音留言** | 錄一段聲音上傳給對方聽（和「語音轉成文字」不同；多數情境可先不做）。 |
| **綁定（客戶與案號）** | 內部人員在系統裡把「這一戶專案」跟「這一位客戶帳號」連起來，連好之後客戶才能用進度頁看自己的案。 |
| **客戶進度頁（唯讀）** | 給屋主只看施工紀錄進度用的畫面，不能改資料、不能留言（除非日後另開功能）。 |
| **Rich Menu（大圖選單）** | LINE 聊天室下面那一排大圖按鈕；可以設成按一下就直接開網頁或開 LIFF。 |
| **排班制** | 員工的班表採每月排定、休假不固定的制度；需到排班系統設定每月哪幾天休。 |
| **標準制** | 員工的班表固定（例如週日休加國定假日），平常不需另排；僅在請假或加班時才需要登記。 |
| **待審核（假勤）** | 有人已送出請假或加班申請，但還沒被主管核准的狀態。 |
| **LIFF** | 在 LINE 裡面開你的網頁的一種方式，網頁程式可以取得使用者的 LINE id（用來對應「哪一位客戶」）。 |
| **Token／權杖** | 一串密碼字元，讓程式確認「這個連結或這支手機可以讀哪一些資料」，不必把整本帳密交出來。 |
| **水電明細表** | 開關、插座、迴路、專用回路等「一覽表」，審水電圖時要拿來和平面／立面互相對照的那張（或那份）。 |
| **+100（圖面標註）** | 常見的「加厚／完成面」尺寸寫法之一，實際代表幾公分要看圖例與比例；重點是壁板／造型要有**可施工的厚度標示**。 |
| **GAS（Google Apps Script）** | 在 Google 雲端跑、常接表單／試算表與觸發排程的小程式；本專案後端 WebApp 在這裡。 |
| **UploadQueue** | 試算表上「待背景處理回報」的佇列列；狀態會從待搬運→待處理→處理中→已完成。 |
| **報價審核器** | 內部用的「工項驗收」網頁（`BudgetAuditor_Standalone_V2`）：連 Firebase 打勾進度、可選顯示單筆金額。 |
| **畫布邊長（Layout Planner）** | 平面可編輯區域的邊長（此工具裡與圖上 cm 1:1）；數字愈大工作區愈大、可捲動範圍愈大。 |
| **報價合併列（LayoutPlanner）** | 名稱單價等條件一樣時，還要看圖上**夠近**才併成表上一列；隔很遠就算尺寸一樣也分開。 |
| **快速報價（清單式）** | 不畫平面圖，用「空間＋坪數＋勾選工項」組出估價明細；適合現場初估，技術規格見 `SPEC/14_INTERIOR_RENOVATION_QUOTATION_SYSTEM_SPEC.md`。 |
| **gviz（網路試算表讀法）** | 試算表在「可匿名讀取」的狀態下，可用網址參數把指定範圍**拉成表格式 JSON**；可 **A 欄＝內用 `id`、B 欄＝單價**，或依**分頁＋A 欄品項**對 **B 單位、C 單價** 寫入品項。 |
| **客戶可見備註** | 管理員在品項上填的說明，**產生報價小表時預設一併顯示**在品名旁邊，方便符合慣用報價格式。 |
| **定量（菜單／估價）** | 數量由菜單**寫死**一個值（幾尺、幾坪、幾式），不隨空間邊長比例重算。 |
| **變量（菜單／估價）** | 數量由**空間的長邊或短邊**、坪等**先算**再依方案乘上**比例%**；寫在方案 JSON 的 `type` 與參數裡。 |

**維護**：新增詞條時保持「一行白話」即可，不必寫論文。
