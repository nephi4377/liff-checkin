## 一句話

會計系統＝**人從 LINE／網頁進來** → **會計後端統一處理** → **寫進幾本試算表**（收支、請款、毛利、附件、薪資、客戶財務等）。

錢分兩種：**已發生 → 立刻記帳**；**還沒付 → 請款審核後，匯款結案才記帳**。

```mermaid

    H2 --> H3["會計選單：收支／請款／審核／待匯／毛利…"]
    H3 --> H4["各功能網頁"]
    L1["開 LINE 進出款群組"] --> L2{"訊息類型？"}
    L2 -->|"打字記帳"| L3["辨識成收入或支出"]
    L2 -->|"傳圖請款"| L4["辨識請款單／走請款流程"]
    L2 -->|"取消／確認按鈕"| L5["改或撤銷剛才那筆"]
    F1["開 LINE 內收支表單<br/>或選單裡的收支登錄"] --> F2["填日期金額案號等送出"]
  end

  subgraph VendorIn["廠商"]
    V1["LINE 綁定過的身分"] --> V2["#請款＋傳圖 或 開請款頁"]
    V2 --> V3["看款項進度"]
  end

  subgraph CustomerIn["客戶"]
    C1["收到確認連結／LIFF"] --> C2["看追加減或收款"]
    C2 --> C3["內容確認／簽名"]
  end

  H4 --> BE["進會計後端"]
  L3 & L4 & L5 & F2 & V2 & V3 & C3 --> BE
```

| 入口 | 典型用途 | 接著走哪條錢路 |
|------|----------|----------------|
| 主控台會計選單 | 日常操作最完整 | A 或 B 或查詢／設定 |
| LINE 群組打字 | 現場快速記已發生收支 | **路徑 A** |
| LINE 群組傳圖＋請款 | 廠商／員工送未付款單 | **路徑 B** |
| LINE／網頁收支表單 | 欄位較完整的已發生記帳 | **路徑 A**（與群組同一條寫帳） |
| 客戶確認頁 | 追加減、收款兩階段確認 | 客戶財務（不直接當日常記帳入口） |

## 3. 兩條錢的路（細步）

### 3.1 路徑 A：錢已經發生（立刻進帳）

```mermaid
flowchart TD
  S0["開始：收支登錄／群組記帳／表單"] --> S1["確認是誰、權限夠不夠"]
  S1 --> S2["整理成標準欄位<br/>收入或支出、日期、金額、案號…"]
  S2 --> S3{"欄位齊全？"}
  S3 -->|否| S3a["回覆缺什麼／怎麼改"]
  S3 -->|是| S4["寫進收支明細<br/>對應月份分頁一列"]
  S4 --> S5["寫完後自動連動"]
  S5 --> S6["有照片 → 存檔＋附件索引"]
  S5 --> S7["有案號 → 寫進案件毛利<br/>明細_X00，列上帶案號"]
  S5 --> S8["若是收入且有案號<br/>→ 建客戶收款待確認"]
  S5 --> S9["記一筆操作稽核"]
```

### 3.2 路徑 B：錢還沒付出（請款 → 審核 → 匯款才進帳）

```mermaid
flowchart TD
  T0["開始：匯款請款頁／LINE 傳圖請款"] --> T1["選或辨識廠商、金額、分攤案號"]
  T1 --> T2["寫進請款申請＝待審<br/>此時不寫收支"]
  T2 --> T3["主管在審核頁核准／退回<br/>可補案號、金額、含稅"]
  T3 -->|退回| T2
  T3 -->|核准| T4["進入待匯款清單"]
  T4 --> T5{"財務怎麼結？"}
  T5 -->|"產中信匯款檔"| T6["只產檔，還不算已匯"]
  T5 -->|"標記已匯款"| T7["這時才寫收支明細"]
  T6 -.->|"匯完再標已匯"| T7
  T7 --> T8["有案號 → 寫進案件毛利"]
  T7 --> T9["可推播通知廠商／記附件"]
  T2 --> T10["款項進度隨時可查狀態"]
  T4 --> T10
```

| 步驟 | 白話狀態 | 寫收支？ | 寫毛利？ |
|------|----------|----------|----------|
| 送請款 | 待審 | 否 | 否 |
| 主管核准 | 待匯款 | 否 | 否 |
| 產中信檔 | 仍待匯（僅產檔） | 否 | 否 |
| 標記已匯款 | 已匯款 | **是** | 有案號則是 |
| 財務捷徑直接建待匯 | 跳過待審 | 仍要等標記已匯才寫帳 | 同上 |

### 3.3 薪資（另一條「還沒付」的支線）

```mermaid
flowchart LR
  P1["出勤／個人頁送薪資請款"] --> P2["薪資審核<br/>可調獎金扣款"]
  P2 --> P3["薪資待匯款"]
  P3 --> P4["產薪資轉帳檔／結案寫收支"]
```

> 廠商待匯款 ≠ 薪資待匯款（匯款檔格式與頁面不同，不要混用）。

## 4. 記帳同一條（群組 ≈ 表單）— 再細一點

人從哪邊進來可以不同，**寫帳與寫毛利是同一套**：

```mermaid
flowchart TD
  E1["LINE 群組：解析文字／按鈕"] --> Norm["整理成標準欄位"]
  E2["表單／選單收支登錄：已填欄位"] --> Norm
  Norm --> Dup{"像不像剛記過的重複單？"}
  Dup -->|要確認| Ask["請人確認要不要再記一筆"]
  Dup -->|通過| Write["寫入收支明細一列<br/>並給這筆一個單號"]
  Ask --> Write
  Write --> Post["寫完後連動"]
  Post --> M["案件毛利：依案號進明細_X00"]
  Post --> Att["有照片：雲端硬碟＋附件索引"]
  Post --> Rec["收入＋案號：客戶收款草稿／待確認"]
  Post --> Log["操作稽核"]
```

| 備註標記（表上常見） | 白話從哪來 |
|----------------------|------------|
| 群組記帳 | LINE 進出款群組 |
| 表單／LIFF | 網頁或 LINE 內收支表單 |
| 廠商請款結案 | 待匯款標記已匯後寫入 |

```mermaid
flowchart TB
  subgraph MarginBook["案件毛利這一本試算表"]
    OV["總覽<br/>每個案子一行：收入合計／支出／毛利／對應哪張明細"]
    D7["明細_700<br/>700～799 多案合在一張，列首有案號"]
    D9["明細_900 … 其他百位同理"]
    Meta["毛利設定（常隱藏）<br/>合約額、基本費用、廠商勾選等"]

    CF2["客戶收款"]
    CF3["客戶財務稽核／待辦／綁定"]
  end

  Ingest["記帳或匯款結案有案號"] --> Pick["依案號百位選明細_X00"]
  Pick --> Row["追加一列：案號＋日期＋收支＋金額…"]
```

| 分頁類型 | 白話用途 |
|----------|----------|
| 總覽 | 找案子、看合計 |
| 明細_X00 | **日常明細唯一入口**（新資料寫這裡） |
| 毛利設定 | 系統記住案的合約與設定 |
| 追加減／收款／稽核／綁定 | 客戶可見財務確認，不是日常記帳本 |
| 舊的「案號_客戶名」 | 歷史遺留；新寫入不再開新的 |

## 7. 試算表／資料本一覽（細）

| 本子（白話） | 主要放什麼 | 誰常寫入 |
|--------------|------------|----------|
| 收支明細 | 已發生收入／支出，按月分頁 | 路徑 A 立刻；路徑 B 匯款結案時 |
| 會計主檔 | 廠商、請款申請、分攤、LINE 聯絡人、薪資請款等 | 名冊、請款、審核、待匯 |
| 案件毛利 | 總覽、明細_X00、毛利設定、客戶財務相關分頁 | 記帳連動、手動補列、客戶確認 |
| 單據附件索引 | 哪一筆帳對哪幾張照片／檔 | 記帳／請款有上傳時 |
| 薪資統計／薪資請款 | 薪資審核與轉帳相關 | 薪資流程 |
| 操作稽核 | 誰做了什麼 | 幾乎所有寫入後 |

**檔案**：單據照片、廠商報價／合約等進雲端硬碟；試算表多半只存連結與索引。

## 8. 人怎麼找功能（選單對照）

| 你要做的事 | 大致入口 | 權限感覺 |
|------------|----------|----------|

| 審核請款 | 請款審核 | 主管核准 |
| 廠商匯款結案／產中信檔 | 廠商待匯款 | 財務 |
| 薪資審核／薪資轉帳 | 薪資審核 → 薪資待匯款 | 主管／財務 |
| 查單據照片 | 單據附件 | 財務 |
| 看／調案件賺賠 | 案件毛利 | 財務 |
| 維護廠商與匯款帳戶 | 廠商名冊 | 財務 |
