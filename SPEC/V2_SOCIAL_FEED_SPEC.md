# 專案主控台 V2 (SocialFeed) 技術規格書-預訂計劃

本規格書定義了「專案主控台 V2」的開發範疇，重點在於實現類似 FB 社群介面的日誌呈現與互動功能。

## 1. 介面與顯示層 (Social UI)

### 1.1 社群貼文牆
- **[NEW] 貼文卡片**: 每一則施工日誌改用卡片式結構，包含作者、時間、內文、多圖網格。
- **[NEW] 多圖遮罩**: 圖片顯示採用 FB 式 1-5 圖網格佈局。當超過 5 張圖時，最後一張顯示疊加層 (例如 `+3`)。
- **[NEW] 日誌刪除**: 增加「🗑️ 刪除」按鈕，僅限 **BOSS 角色**或 **原貼文者** 可點擊。

### 1.2 互動功能 (留言系統)
- **留言預覽**: 預設顯示最新的 3 則留言。
- **查看更多**: 留言超過 3 則時，顯示「查看較早的留言...」按鈕，點擊後展開其餘內容。
- **輸入框權限**: 僅限設計師 (BOSS) 與該專案成員可進行留言。

## 2. 邏輯與功能件 (Interaction & Logic)

### 2.1 頭像管理 (SPEC 1.D.1)
- **頭像快取**: 實作 `getAvatar()` 邏輯。
    - 優先從 `sessionStorage` 讀取員工頭像 URL。
    - 若無頭像，則根據 UserID 生成固定背景色，並顯示「姓名首字」。
- **效能緩衝**: 快取機制可減少 CDN 圖片請求，並加速頁面初次渲染。

### 2.2 錯誤重試機制 (SPEC 1.D.5)
- **樂觀更新 (Optimistic Update)**: 使用者點擊發佈留言後，UI 立即渲染一個「半透明(Pending)」狀態的留言。
- **異常處理**: 若 API 返回失敗或網路連線異常，系統需：
    - 將留言標記為紅色失敗狀態。
    - 顯示「⚠️ 重試」按鈕，允許使用者重新發送該條留言。

### 2.3 分頁與載入 (Optimization)
- **無限滾動**: 在 V2 社群牆模式下，日誌載入需具備無限滾動功能。
- **背景刷新**: 溝通紀錄 (Chat) 區塊需保持輕量化背景更新 (每 10 分鐘)。

## 3. 環境隔離性

### 3.1 獨立路徑
- 所有 V2 改動必須限制在 `modules/projects/v2/` 資料夾內。
- 專門使用 `managementconsole_v2.html` 搭配 `js/main_v2.js`。
- 不得修改 `modules/projects/` 的 V1 正式版檔案。
