# Code Advisor 白話提問指南

## 原則

1. **對使用者用結果與感受問**，題目不出現函式名、變數名（改法在表格「改法摘要」，AskQuestion 選項也要白話）
2. **釐清意圖**：一次最多 **3～5 題**；每題 A～D + E 其他
3. **能從 code／diff 推斷就先審**；真缺業務決策才問
4. 全篇確認區 **不超過 25 行**

## 審碼前：缺資訊才問（3～5 題）

| ❌ 不要 | ✅ 改成 |
|--------|--------|
| retry 要 exponential backoff 嗎？ | 失敗時你希望：**立刻再試／隔幾分鐘再試／只記錄不重試** |
| 這欄位對字典 `order_total` 嗎？ | 畫面上的「總金額」應該：**含運費／不含運費／跟報價單一致就好** |
| 要抽成 service 嗎？ | 這次你希望：**只修眼前 bug／順便整理讓以後好改** |

## 審完：請總監選要改哪幾項

**AskQuestion**（`allow_multiple: true`）選項 = 白話一句，格式：

`A [會壞] 使用者填的資料可能存錯 — 我幫你對齊欄位`

固定加：`all` 全部套用、`none` 先不改只要說明。

**複選若不好用**（工具不支援或總監說答不出來）→ 改文字回覆：

```markdown
請回：**A**、**A+C**、**全改** 或 **先不改**
```

## 事後調整（先審完再問）

```markdown
我假設你希望 **（白話一句）**。要改嗎？
- A) 維持
- B) …
- E) 其他
```
