# 別人的 Cursor 規則範本（精選 8 份）

## 這是什麼？

別人寫好、放在 GitHub 的「叫 Cursor 怎麼寫程式」的說明稿。  
我們挑了 **8 個 `.mdc` 檔**，放在 **`vendor/` 資料夾裡（直接是檔案）**，**內容已翻成繁體中文**。

## 為什麼放 skill，不整包貼進專案？

- **Skill**：像工具書，需要翻才打開，不會每次聊天都塞一大包。
- **`.cursor/rules/`**：像你訂的家規，要常常遵守的才放那裡（你已有「白話總規則」「上傳流程」）。

## `vendor/` 裡有什麼？（點檔名就開）

| 檔名（都在 vendor 根目錄） | 大概什麼時候看 |
|---------------------------|----------------|
| `vue3-composition-api-....mdc` | 改 Vue 3、spa 外殼 |
| `vue.mdc` | 一般 Vue |
| `html-tailwind-css-javascript-....mdc` | HTML + Tailwind 頁面 |
| `javascript-typescript-code-quality-....mdc` | JS 寫乾淨一點 |
| `code-guidelines-....mdc` | 通用寫法習慣 |
| `code-style-consistency-....mdc` | 跟現有程式碼風格一致 |
| `typescript-vuejs-....mdc` | 有 TypeScript 的 Vue |
| `chrome-extension-dev-....mdc` | 瀏覽器／桌面周邊 JS（參考） |

**Google 試算表後端（GAS）這裡沒有範本**，仍以 `SPEC/` 為準。

## 和專案規則打架時

以 **`.cursor/rules/00-general-plain-language.mdc`**（白話、簡答）和 **SPEC** 為準，範本只當參考。
