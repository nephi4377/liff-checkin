# 添心設計公司 — 業務與技術實務標準資料表 (v1.3)

本文件儲存公司內部估算、報價、工法、系統配置、自動化判定關鍵字及資料庫欄位之具體業務數據與技術快照，供 AI 助理依需求按需載入（On-demand Loading）。

---

## ── L1：商業與專案定位 ──
* **主攻業務**：系統與初談以**「新成屋大樓（新成屋）」**為主。
* **服務區域約束**：**不另收車馬費**以維持計價單純；但若建案地點過遠（例如雲林及更遠地區）則**不承接專案**。
* **商業授權限制**：Gemini 嚴禁自行執行生產環境（PROD）部署或 `clasp deploy`。自動化部署與發布指令（`upload.bat` / `deploy.bat`）必須由總監明確下令「部署」或「發布」後方可執行（SafeToAutoRun: true）。
* **團隊規模假設**：本工作區為**單人開發**，省略團隊 Review、遠端 Git 多分支衝突與團隊協作假設。

## ── L2：設計空間分區與推估模型 ──

### 1. 空間分區結構
* **四大核心分區**：
  1. **客餐廳** (`lk`)：客廳與廚房合併為一區。
  2. **主臥** (`master`)
  3. **次臥** (`bed2`, `bed3`, `bed4` 等)：依房間數逐間分列。
  4. **全室** (`all`)：不屬於單一房間的共用項目（天花板、地坪、面漆、防護）。
* **顯示與產出順序**：固定為 **`客餐廳 → 主臥 → 次臥（依臥室序） → 全室`**。

### 2. 房型坪數分配模板
當系統依據業主輸入的「總坪數（不含公設）」進行推估時，其空間分配規則如下：
* **二房一廳一衛**：分配 2 臥室、客廚、1 衛。
* **三房二廳二衛**：分配 3 臥室、客廚、2 衛。
* **四房二廳二衛**：分配 4 臥室、客廚、2 衛。
* **平面幾何反推**：各區以 **`長邊 : 短邊 = 6 : 4`** 的長方形比例反推槪略長寬（cm），並換算為**「台尺」**（1 尺 ≈ 30.3 cm），以供櫃體延尺估算。

## ── L3：工項實務施作與工法標準 ──

### 1. 水電與弱電工法
* **新大樓插座估算（無底數按需累加）**：
  * **不設基礎底數**（無任何變動則新增數量為 0）。
  * 廚房電器櫃區：估算新增 **`2 點`** 專用迴路（預設採用 **`5.5mm² 絞線`**）或局部移位。
  * 客廳電視牆：配合電視櫃位置進行高度與孔數微調移位（預設 **`3 點`**）。延長原有 Cat6 網路線與同軸線，提案時主動詢問業主是否需拉直連新線。
  * 各房床頭與書桌：局部增設或移位（**每區預設 `2 點`**，如主臥 2 點，次臥各 2 點）。
  * 冷氣排水：若建商預留高度不足，**必須進行牆面或地面敲打埋管**。
  * 嵌燈迴路：單一迴路控制上限為 **`10 盞`**，預設使用 **`12W 嵌燈`（色溫 4000K 自然光為主）**。
* **中古屋/舊屋大樓全室翻新水電**：
  * 採**「空間計價」**，每個獨立空間 **`12,000 元`**（客餐廳計為 2 個空間，即 24,000 元）。

### 2. 木作與系統櫃體
* **櫃體預設工法**：預設以 **`系統櫃`** 為主。若改為木作單價不變，但因人工收邊質感不同，需提醒業主。
* **櫃體收邊**：一律由系統櫃廠商現場裁切系統板進行**「速成收邊」**。
* **基礎五金**：櫃體單價已內含**「基本進口緩衝鉸鏈、普通三節軌抽屜」**，其餘拉籃、旋轉衣架等特殊五金另行計價。
* **天花板角材與板材**：平釘天花板預設使用 **`集成角材 (LVL) + 台灣麗仕（或同級）矽酸鈣板`**。

### 3. 材料損耗與清運
* **石材檯面**：餐櫃或電視牆檯面**獨立拆出「石材工程」大類**，由石材廠商報價，我們會**加收監工費**。
* **耗損係數**：
  * 地磚損耗：實鋪坪數乘以 **`1.1`**
  * 木地板損耗：實鋪坪數乘以 **`1.15`**
* **廢料搬運與垃圾清運**：在保護/清運大類中**獨立列「一式」計價**，不攤提在材料單價中。

### 4. 一般裝修估價單標準工項與新舊屋判定
* **估價單編寫原則**：
  1. **依施工順序與工種分章**：避免工種穿插，方便現場管理。
  2. **空間分區明細**：木作、系統櫃等客製化工程依「玄關、客廳、各臥室」分區，便於對圖。
  3. **規格與備註透明**：註記材料品牌、厚度、尺寸界線（如麗仕矽酸鈣板、Panasonic星光等）。
* **標準工種與施工排序**：
  1. **保護工程**：室內地坪+廚具保護、門框防撞保護、室外公設保護、粗清垃圾搬運。
  2. **拆除工程**（舊屋專屬）：拆除舊裝潢、地磚、牆面或廚具衛浴。
  3. **水電工程**：插座開關新增/位移、高功率專用迴路、Cat6網路與同軸線路延伸。
  4. **泥作工程**（舊屋專屬，新屋極少用）：牆面地坪打底、浴室防潮防水三道、磁磚鋪貼。
  5. **木作工程**：平釘天花板、包管窗簾盒、木作隔間牆、造型木作與壁板。
  6. **系統櫃工程**：各區客製衣櫃、鞋櫃、收納高櫃等。
  7. **油漆工程**：批土二至三道、矽酸鈣板水泥漆/乳膠漆、填縫收邊。
  8. **收尾與清潔**：粗清/竣工細清、崁燈與軌道燈安裝、局部點補修補。

## ── L5：系統與儲存配置常數 ──

### 1. 本機與儲存鍵名
* **快速報價系統配置**：`localStorage` 鍵名為 **`qr_estimator_settings_v2`**，存儲 `itemDefs` 與 `schemeMenus`。
* **驗收審核系統配置**：`localStorage` 鍵名為 **`fb_audit_config`**（雲端 Firebase/GAS 配置）、**`audit_cache_{案號}`**（離線進度快取，快取期 24 小時）。
* **Firebase 雲端路徑**：
  * 報價/驗收工項：`quotations/{projectNo}`
  * 施工回報上傳：`reports/{projectId}/{batchId}/{timestamp}_{idx}_{rand}_{safeName}`
  * 失敗日誌收集：`failure-report/{projectId}/failure-{timestamp}.json`

### 2. 面積與計價常數
* **面積單位換算**：1 坪 = **`3.305785124 ㎡`**。
* **金額捨入模式**：小計、設計費、服務費及稅額均採用 **`HALF_UP`（四捨五入）至整數元**。
* **試算表對齊 Gap 項目**：
  * 鞋櫃 / 收納櫃 $\rightarrow$ 對照雲端 `木作工程` 分頁下 `開門高櫃-D40H240內` 列。
  * 衣櫃 $\rightarrow$ 對照雲端 `木作工程` 分頁下 `開門高衣櫃-D60H240內` 列。

## ── L6：報價與施工回報行政流程規範 ──

### 1. 報價與費用
* **設計與監工費率**：本快速報價專案**「免收設計費及監工費（工程服務費）」**。
* **稅率與開立**：報價單預設為 **`未稅`**（發票需另計 5% 營業稅，於總價下方單獨列出稅額）。
* **自訂項目金額上限**：設計師/助理可直接增減自訂工項，**無金額上限限制**，但重大項目必須在備註加註原因。
* **版本控管方式**：直接在原報價單上修改並覆蓋，但定稿前助理必須**手動匯出 Excel 明細作為本機備份**。
* **免責聲明與簽認**：PDF 底部必須附帶固定免責聲明：*「此報價為估算參考，實際金額以正式合約與施工圖為準」*，**不需客戶當場簽名**。

### 2. 施工回報與相片壓縮
* **現場照片壓縮參數**：預設使用 **`maxSizeMB: 0.4`**、**`maxWidthOrHeight: 1280`**，且提供使用者可於前端手動關閉壓縮之勾選框。
* **極速上傳併發**：`PHOTO_UPLOAD_CONCURRENCY: 5`，上傳至 Firebase 成功後背景透過 GAS 佇列搬運至 Google Drive 暫存。
* **多裝置同步衝突**：採 **`Last Write Wins`**（最後同步者覆蓋雲端）。

### 3. 權限分級顯示 (驗收表)
* **未滿 4 級**：隱藏工項金額 `price` 與表頭總價卡；JSON 匯出時刪除 `price` 欄位鍵。
* **4 級以上**：顯示單筆 `price` 與合計卡。
* **5 級以上**：開啟 JSON 匯出按鈕。

## ── L7：統一結案防呆與對帳機制 ──
* **結案權威真相**：以 Google 試算表 `案場資料`.`專案狀態` 的 **`已結案`** 為公司級案場生命週期的唯一事實來源。
* **Firebase 標記**：當專案結案時，Firebase `quotations/{案號}` 的 `is_closed` 欄位將同步被標記為 `true`，以供驗收表 UI 一致呈現。
* **防呆限制**：
  * **進度與結案分離**：結案動作**不得**自動將 Firebase 工項進度改為 100%，兩者狀態獨立。
  * **對帳節流**：結案 API 對帳 Firebase 之快取失效時間為 **`1 小時`**（快取鍵名 `closure_fb_sync_{案號}`），且單次批次請求最大處理案數限制為 **`8 案`**。
