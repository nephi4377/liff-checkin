# human comfortable

## 核心目標

讓**完成後的成品**直覺好用，減少使用者的**腦力**（不用猜）與**眼力**（不用找）負擔。

**管**：真人會看、點、填、讀的介面。  
**不管**：Agent 回覆格式、log／JSON 當預設畫面。

## 三大原則（定義本 skill）

| 原則 | 白話 |
|------|------|
| **看哪裡很清楚** | 視覺上的舒服 |
| **要做什麼很簡單** | 操作上的直覺 |
| **做錯了不挫折** | 心理上的安定 |

另加 **設備上的貼心**：手機、小螢幕、字體與按鈕要夠大、夠好點。

---

## 1. 看哪裡很清楚（視覺）

- **不要塞滿**：像雜貨店一樣亂；寧可空一點，不要擠滿。元件之間**留白**，有距離。
- **層級分明**：最重要的（確定、存檔）**最大最明顯**；次要（取消、重設）縮小或換成次要樣式。同區**只有一個主按鈕**。
- **顏色有意義**：避免刺眼亮色；紅＝危險／刪除，綠＝成功／完成（須搭配文字，不只靠色）。
- **預設先摘要**：不要一開就是整表、整棵樹、整段 JSON；明細收起、分頁或第二層。
- **藏少用的**：「這輩子只點一次」的大功能改進階、設定或更多裡，別佔主畫面。

**每頁結構（必備）**

```
[標題：這頁在做什麼]
[一行狀態或說明（可選）]
[主操作區：一個最突出的主按鈕]
[內容：摘要／卡片優先]
```

CODING 網頁：配色、圓角、陰影依 `Premium-UI-Design`；文案依 `00-general-plain-language`。

---

## 2. 要做什麼很簡單（操作）

- **按鈕要像按鈕**：形狀、hover、按下狀態都要讓人知道「可以點」。
- **少點幾下**：最重要功能直接擺在畫面上，不要藏三層選單；能自動帶入的（日期、合理預設值）就幫填。
- **即時反應**：點下去立刻有變化（轉圈、按鈕變深、disabled +「處理中」），不要讓人懷疑當機。
- **四態必做**：載入中｜沒資料（說明 + 建議動作）｜失敗（人話 + 怎麼修）｜成功（簡短提示，不必大跳窗）。
- **表單**：可見標籤（placeholder 不能當標籤）；錯誤在欄位下方；送出中防連點。
- **表格**：欄名白話中文；不要一次灌爆；長表分頁或捲動、表頭可讀。

---

## 3. 做錯了不挫折（心理）

- **人話提示**：禁止「Error 500」「非法參數」；改說「電話格式不對，請檢查」或「網路不穩，請稍後再試」。
- **反悔機制**：刪除、結案、覆蓋等要有**確認**（寫清後果）或**還原**；確認框預設焦點在取消。
- **少打斷**：非必要不要用巨大視窗擋全頁；用旁邊小氣泡、橫條等輕量提示。

---

## 4. 用什麼都好用（設備）

- **字體**：功能說明、標籤不能太小；要讓一般使用者（含長輩）讀得清。
- **觸控**：手機版按鈕夠大、間距夠，避免點 A 卻按到 B。
- **小螢幕**：仍讀得清、主按鈕仍找得到（若此成品會用手機看）。

---

## 5. Agent 執行檢查清單

**寫完或改完給人用的介面後**，逐項自檢（有一項否就繼續改）：

- [ ] **一眼看過去**：使用者知道現在該點哪裡嗎？
- [ ] **操作預測**：有沒有任何功能必須翻說明書才會用？
- [ ] **錯誤處理**：填錯時，介面會溫柔引導修好嗎？（欄位旁、人話）
- [ ] **多餘資訊**：少用的 giant 功能是否已藏起？
- [ ] **四態齊全**：載入／空／錯／成功都有、且是人話
- [ ] **不嚇人**：無無謂全屏跳窗；危險操作有確認；桌面啟動不閃黑窗（若適用）
- [ ] **預設無工程垃圾**：畫面預設沒有 log、stack、raw JSON

---

## 禁止（常見不友善原因）

- 滿版按鈕、滿版表格、沒有主從。  
- 按了沒反應。  
- 錯誤只顯示錯誤碼。  
- 成功／失敗只靠顏色。  
- `alert()` 或巨大 modal 當唯一回饋。
