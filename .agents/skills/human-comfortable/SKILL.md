# human comfortable

> **human comfortable** = 預設讀者／操作者是**一般使用者或未來的自己**，不是除錯主控台、不是 log 檔、也不是只有工程師才看得懂的畫面。

## 何時用這份 skill

- 新增或改版**任何會被人看到**的東西（UI、郵件、通知、錯誤、報表、說明、Agent 回覆格式）。
- 使用者抱怨「看不懂、找不到、被嚇到、不知道有沒有在跑、不知道下一步」。
- 在「技術正確」與「人用得順」之間取捨時，**預設選人用得順**（技術細節可收合、放進階區或 log）。

## 核心原則（通用）

1. **先結論，再細節**
  第一屏／第一句回答「現在怎樣、該做什麼」；長清單、堆疊追蹤、原始 JSON 預設收合或分頁，不要一開就淹沒人。
2. **白話優先**
  按鈕、狀態、錯誤用日常中文；英文或路徑可保留，但加一句「這代表什麼／你可以怎麼做」。與 CODING `00-general-plain-language` 並用，不衝突。
3. **少驚嚇**
  - 破壞性操作要明確標示並確認。  
  - 不要無預警彈窗、全螢幕閃爍、突然音效。  
  - 失敗時說**人話原因 + 建議下一步**，不要只丟錯誤碼或整段 stack。
4. **讓人知道「正在發生什麼」**
  超過約 1～2 秒的操作：進度、狀態列、或「背景執行中，可關閉視窗」；完成或失敗要有**可見**回饋（不必一定要 modal）。
5. **掃一眼就懂**
  - 嚴重度用一致圖示／色塊（成功／警告／錯誤）。  
  - 數字用合適單位（KB/MB、% 、日期本地化）。  
  - 長清單截斷並提示「展開看全部」。
6. **一條明路**
  錯誤或空狀態要告訴人：**你可以做什麼**（重試、檢查設定、聯絡誰、去哪個畫面），避免死胡同。
7. **預設安全、可復原**
  能 undo 就 undo；不能 undo 就二次確認並說清楚後果。

## 依產物類型的檢查（擇用）


| 類型                  | 多問自己一句                                                   |
| ------------------- | -------------------------------------------------------- |
| **網頁／後台**           | 空狀態、載入中、錯誤是否同一套語氣？表單錯誤是否指到欄位？                            |
| **API／後端回傳給前端**     | 除 `success` 外，是否有**使用者可顯示**的 `message`（與工程用 `debug` 分開）？ |
| **CLI／腳本（給人跑）**     | 預設輸出是否簡短可讀？`-v`／`--json` 給進階即可。                          |
| **桌面程式（Windows 等）** | 雙擊是否仍閃 cmd？子程序是否多開黑窗？見下方「Windows 桌面」                     |
| **報表／匯出**           | 是否有「一行結論 + 摘要表」？細節是否另頁／另檔？                               |
| **排程／背景工作**         | 結果是否寫到**人找得到**的地方（UI、通知、約定路徑），不只寫在無人看的 log？              |
| **Agent／助理回覆**      | 先結論；步驟用編號；避免無謂長篇與術語牆。                                    |


## 實作檢查表（改版前快速掃）

- 第一次打開能否在 **10 秒內** 知道「這是什麼、我該做什麼」？
- 最壞情況（錯誤、空資料）是否仍看得懂、知道下一步？
- 長時間操作是否有進度或「還在跑」的訊號？
- 技術細節是否預設隱藏，需要時才展開？
- 破壞性動作是否有確認與後果說明？
- 文案是否像對人說話，而不是對機器 dump？

## Windows 桌面（可選模式，非全 skill 範圍）

僅在**桌面工具**且目標是「一般使用者雙擊就用」時：

- 預設避免黑窗：`pythonw`、VBS 隱藏啟動、子程序 `CREATE_NO_WINDOW`；排程優先直接叫 `pythonw`，避免 `cmd /c *.bat` 除非必要。
- ## 背景 log 寫約定路徑（如 `%LOCALAPPDATA%\…\runtime.log`），UI 可 tail，前綴區分「背景／手動」。

## 與其他 skill／規則

- **白話**：CODING `00-general-plain-language`、`SPEC/GLOSSARY_ZH.md`。
- **網頁視覺**：`Premium-UI-Design`（Tailwind 等）；桌面 Tk/CTk 用卡片、色塊、可捲動文字區即可，不必硬套網頁框架。
- **工程師要看細節**：用 log、debug 模式、進階分頁；**不要**把預設畫面變成工程師專用。
