# 多任務角色分工表

父代理 = 協調者。子代理在背景跑，完成後由協調者合成。

## 總表

| 角色 | 誰 | 做什麼 | 不做什麼 |
|------|-----|--------|----------|
| **協調者** | 父代理 | 理解使用者目標；拆成主工＋可並行唯讀工；派背景子代理；等通知；白話合成結果；決定下一輪要不要再派 | 不自己重複探索者已查過的 code；不在子代理已改檔後前景再改同一處；不整段轉貼子代理報告 |
| **探索者** | `explore` 子代理 | 唯讀搜尋、讀 SPEC、grep、回報「現況＋缺口＋建議方向」 | 不改檔；不部署；不開本地站 |
| **實作者** | `generalPurpose` 子代理 | 依派工說明改檔、實作功能、補規格文件 | 不擅自 commit／部署；不跳過協調者直接對使用者長篇報告（回傳給父代理即可） |
| **自測者** | `generalPurpose` 子代理 + Runbook | 依 `@Cloud-Agent-Runbook` 開本地站；靜態檢查；瀏覽器驗證；修小問題（文案、明顯 UI） | 不大改架構；不部署正式站；測不過時回報根因給協調者，不無限試錯 |
| **審查者** | `bugbot`／`security-review`（選用） | 使用者明確要求時，對 diff 做品質或資安審查 | 不實作功能；不代替自測者跑本地站 |

## 常見組合

| 使用者要什麼 | 建議派工 |
|--------------|----------|
| 先搞清楚再改 | 探索者 →（有結論）→ 實作者 |
| 改完要驗 | 實作者 → 自測者 |
| 大規格＋實作並行 | 探索者（查現況）‖ 實作者（寫文件草稿）— 僅文件與查詢可並行 |
| 改完＋順便審 | 實作者 → 自測者 ‖ 審查者（使用者有要求時） |

## 協調者決策速查

```
收到使用者需求
  ├─ 小、單步？ → 自己做，不派子代理
  ├─ 要先懂現況？ → 派探索者（背景）
  ├─ 已知怎麼改？ → 派實作者（背景）
  ├─ 改完要驗？ → 派自測者（背景，附 runbook）
  └─ 要 review？ → 使用者明確說了才派審查者
```

## 回報責任

| 角色 | 回報對象 | 內容 |
|------|----------|------|
| 子代理 | 父代理（協調者） | 技術細節可完整；標出結論、檔案、阻塞 |
| 協調者 | 使用者 | 白話、一句結論優先；見 SKILL.md「對使用者回報」 |
