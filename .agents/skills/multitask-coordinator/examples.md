# 多任務協調範例

以下改寫自 CODING 專案真實工作流（會計快取、流程圖、自測修復）。

---

## 範例 1：會計快取 — 探索 + 規格對照

**使用者：**

```text
@multitask-coordinator 會計快取跟身分傳承有沒有對上 SPEC 18、19？順便標缺口
```

**協調者（派工前，一句）：**

```markdown
我會派背景同事唯讀查前端暫存、後端快取、身分怎麼傳，對照 SPEC 18、19 後用白話回你。
```

**派工（內部，不給使用者看細節）：**

- 探索者（背景）：讀 `SPEC/18_*`、`SPEC/19_*`；搜尋會計 bootstrap、操作者身分、HUB 傳承相關程式；列「一致／缺口／建議」

**協調者（子代理完成後，對使用者）：**

```markdown
## 一句結論
大方向跟 SPEC 18、19 一致；有 2 處文件沒寫到實際保留天數差異。

## 結果
1. **分頁暫存**：操作者身分、會計主檔快取 — 行為符合「關分頁前有效」
2. **HUB 傳承**：從父層 iframe 帶身分進會計 — 有做，跟 SPEC 19 流程一致
3. **缺口**：本機長期暫存 3～7 天在程式裡有分版本，SPEC 18 只寫區間沒寫觸發條件
4. **缺口**：後端快取 6 小時上限 — 程式有，文件未標「平台硬上限」

## 下一步
- A) 只記缺口，我自己改 spec
- B) 幫我補進 SPEC 18 對照表
- C) 直接派同事畫流程圖（範例 2）
```

---

## 範例 2：流程圖 — 實作者補 SPEC 20

**使用者：**

```text
多任務協調：把會計快取跟身分傳承畫成流程圖，白話優先，程式名放文末
```

**協調者（派工前）：**

```markdown
我會派同事撰寫 SPEC 20 流程圖（mermaid + 白話表），完成後給你摘要。
```

**派工：**

- 實作者（背景）：新增／更新 `SPEC/20_會計快取與身分傳承流程圖.md`；圖用白話；程式對照放文末附錄

**協調者（合成後）：**

```markdown
## 一句結論
已補上 SPEC 20：整體架構圖、HUB→會計身分流程、快取讀寫順序，都用白話。

## 結果
- 檔案：`SPEC/20_會計快取與身分傳承流程圖.md`（v1.2）
- 含 5 張 mermaid 圖 + 分頁／本機／後端快取對照表
- 程式名稱集中在文末「函式／模組對照表」，正文不堆術語

## 下一步
- A) 你先讀圖，有錯再叫我改
- B) 派自測者對照本地站實際行為（範例 3）
```

## 範例 3：自測修復 — 自測者 + 小修

**使用者：**

```text
當大腦派子代理：本地站開起來，看會計頁身分有沒有從 HUB 帶過來，不對就修小問題
```

**協調者（派工前）：**

```markdown
我會派同事依 Runbook 開本地站、驗 HUB→會計身分；能修的小問題直接修，大的回報我。
```

**派工：**

- 自測者（背景）：載入 `@Cloud-Agent-Runbook`；`serve` 開站；瀏覽器走 HUB 進會計；記錄現象；若僅 session 鍵名或顯示文案可小修

**子代理回協調者（內部摘要）：**

- 本地站正常；身分有帶入；儀表板標題仍顯示舊文案「未登入」— 已改為顯示操作者暱稱

**協調者（對使用者）：**

```markdown
## 一句結論
本地驗過：從 HUB 進會計，身分有正確帶入；順手修了頁首還顯示「未登入」的文案。

## 結果
- 本地站 `127.0.0.1:3000`：HUB → 會計路徑 OK
- 小修：會計頁頁首改顯示目前操作者暱稱（未部署）

## 下一步
- A) 滿意，先不部署
- B) 要我走 deploy-runbook 上線（你要明說「部署」）
```

## 反例：協調者自己做又派 explore

**錯：**

協調者自己 grep 三輪，又派探索者「幫我搜會計快取」，完成後把探索者 80 行報告整段貼給使用者。

**對：**

協調者一句結論 → 只派探索者 → 結束回合 → 收到通知後合成 5 條白話結論。
