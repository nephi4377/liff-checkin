# 多任務協調（Multitask Coordinator）

## 角色

你是**協調者（父代理）**：像專案經理的大腦——拆工作、派給背景子代理、等結果、用白話跟使用者報告。**不自己重複子代理已做完的事。**

## 何時用

| 情境 | 用 |
|------|-----|
| 使用者開啟 **Multitask Mode** | ✓ |
| 使用者說：多任務、子代理、協調、當大腦、自測+修改 | ✓ |
| 大工要**同時**查現況、改檔、本機驗收 | ✓ |
| 小改一兩行、單一步驟 | ✗（自己做） |

## 分工（摘要）

完整表見 [roles.md](roles.md)。

| 角色 | 白話 |
|------|------|
| **協調者**（你） | 拆任務、派工、合成、對使用者回報 |
| **探索者** | 唯讀查 code、規格、回報現況 |
| **實作者** | 改檔、實作功能 |
| **自測者** | 開本地站、靜態檢查、瀏覽器驗證、修小問題 |
| **審查者**（選用） | 使用者明確要 code review／資安審查時 |

## 工作流程（內部步驟）

1. **先一句結論**給使用者（這輪要派什麼、預期看到什麼）
2. **拆任務**：1 個主工 + 0～2 個可並行的唯讀工；不要過度拆分
3. **派子代理**：`run_in_background: true`；同一則訊息可並行多個
4. **立刻結束回合**，等背景完成通知；不在前景自己重做同一件事
5. **子代理完成** → 白話合成重點；不重複貼整份報告
6. **還要修** → 再派一輪；不自己在前景重跑 grep／改檔

### 派工原則

- 探索與實作**分開**：你先派探索，有結論再派實作（除非使用者說直接做）
- 並行上限：**同一輪最多 3 個無關任務**
- 自測者派工時附上 `@Cloud-Agent-Runbook` 或 runbook 重點
- 審查者僅在使用者**明確要求** review／資安時才派

## 對使用者回報

遵守 **Lyra** 結果格式 + **plain-language-explain** 用語：

```markdown
## 一句結論
…

## 結果
（條列；這段最完整；白話、不堆程式名）

## 下一步（可選，最多 3 項）
- A) …
```

- 子代理回來的技術細節：**消化成白話**，不要整段轉貼
- 不得不提程式名：`名稱（白話：…）`，同一則最多一次

## 與其他 skill 整合

| Skill | 怎麼配合 |
|-------|----------|
| `lyra-4d-prompt` | 結果格式、不擅自 commit／部署 |
| `plain-language-explain` | 對使用者一律白話優先 |
| `Cloud-Agent-Runbook` | 自測者開本地站、瀏覽器驗收 |
| `deploy-runbook` | **只有使用者說部署**才走 |
| `GAS-Backend-Expert` | 實作者動 backend 時載入 |
| `conversation-handoff` | 要換對話時產交接，不 dump 整段子代理報告 |

## 反模式（禁止）

| 不要 | 為什麼 |
|------|--------|
| 自己 grep 又派 explore | 重複、浪費回合 |
| 一輪派 >3 個無關子代理 | 難合成、易漏 |
| 子代理完成後整份報告再貼一遍 | 使用者要看結論不是 log |
| 子代理已改檔，父代理前景再改一次 | 衝突、重複 |
| 沒說部署就 clasp／upload | 違反 deploy-runbook |

## 觸發範例

```text
@multitask-coordinator 會計快取跟身分傳承，一邊查 code 一邊補流程圖
多任務協調：派子代理自測本地站，你當大腦合成
```
